Pourquoi ce projet ?

3 / 18
Contexte & problème identifié
"Chaque pôle suit ses indicateurs de son côté, avec ses propres méthodes."
📁

Données dispersées

Fichiers Excel isolés, échanges informels, notes terrain. Chaque entité a ses propres supports, incompatibles entre eux.

🔍

Pas de vision globale

Impossible de comparer les entités, de voir les tendances ou d'identifier rapidement les écarts de performance.

🚧

Information bloquée

Les remontées terrain ne sont pas formalisées. Le pilotage repose sur la mémoire et l'expérience des personnes.

Accompagnement BPI France en cours

Une démarche de management visuel est engagée : indicateurs simples, visibles, régulièrement mis à jour. Ce projet en est le support concret.

L'objectif du projet

5 / 18

Mission principale

Mettre en place, avant juin 2026, un site intranet permettant la saisie, la centralisation et la visualisation des indicateurs de performance — simple, utilisable au quotidien.

1

Structurer les indicateurs — définir une base commune en lien avec la démarche BPI France

2

Développer le site — page d'accueil, pages par entité, formulaires de saisie, tableaux de bord

3

Gérer les rôles — opérateur, responsable, direction : chacun voit et saisit ce qui le concerne

4

Assurer la simplicité — prise en main en moins d'une journée, saisie en quelques minutes

5

Prévoir l'évolution — base solide, facile à enrichir sans refonte complète du site

Comment les briques communiquent entre elles
🖥️

Le navigateur (Chrome, Edge…)

Ce que voit et utilise chaque employé. Affiche les formulaires, tableaux de bord et indicateurs.

↕ envoie / reçoit des données via le réseau interne
⚙️

Le serveur Node.js

Programme tournant en arrière-plan sur un PC du réseau. Reçoit les saisies, vérifie les droits, stocke les données.

↕ lit et écrit dans le fichier de données
💾

La base de données (JSON)

Fichier texte structuré contenant toutes les données : utilisateurs, indicateurs, paiements, incidents…

Réseau interne uniquement Aucune donnée sur Internet Démarrage en 1 clic

16 / 18
Points forts techniques
🔒

Sécurité des accès

Compte personnel pour chaque utilisateur. Le serveur vérifie les droits à chaque action. Code admin pour les opérations sensibles.

💾

Données conservées

Toutes les saisies sont historisées. Consultation semaine par semaine, analyse des tendances dans le temps.

🔄

Import depuis Excel

Le serveur lit automatiquement les fichiers Excel des dossiers partagés réseau (Z:\, W:\, X:\) sans manipulation manuelle.

🛡️

Protection anti-abus

Limite automatique à 60 saisies par minute par poste. Protège contre les erreurs de manipulation en masse.

🧪

Tests automatisés

5 programmes vérifient le bon fonctionnement : syntaxe, authentification, utilitaires, logique métier.

🖥️

Démarrage simplifié

Un double-clic sur DEMARRER-SERVEUR.bat lance l'ensemble du système. Aucune compétence technique requise.
